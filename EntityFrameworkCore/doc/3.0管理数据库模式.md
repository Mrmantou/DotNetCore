#### 管理数据库架构(Managing Database Schemas)

EF Core提供了两种方式来保持EF Core模型和数据库架构的同步。在这两中之间的选择取决于到底是使用EF Core模型还是数据库架构作为真实之源。

如果使用EF Core模型作为真实之源，使用迁移。在对EF Core模型进行修改后，这种方法逐步地将相应的模式更改应用到数据库，以便与EF核心模型保持兼容。

如果使用数据库架构作为真实之源，使用反转工程。这种方式允许通过将数据库模式反向工程到EF Core模型以便进行DbContext和实体类型。

>**Note:**
>
> [create and drop APIs]也可以从EF Core模型创建数据库架构。然而，它主要用于测试、原型化和其它可以接受删除数据库的场景。
